# عمارة النظام

## التصميم الشامل لنظام قبو الذكاء الاصطناعي المعزز

هذا الملف يشرح البنية المعمارية الكاملة للمشروع والمكونات الأساسية.

---

## 1️⃣ نظرة عامة على العمارة

### المبادئ الأساسية

المشروع مبني على ثلاثة مبادئ أساسية:

1. **التنظيم الهرمي**: تنظيم المحتوى في فئات وأقسام واضحة
2. **القابلية للتوسع**: إمكانية إضافة محتوى وميزات جديدة بسهولة
3. **سهولة الوصول**: واجهة سهلة التصفح واستخراج البيانات الآلي

### الرؤية المعمارية

```
┌─────────────────────────────────────────────────────────┐
│          طبقة المستخدم (User Layer)                    │
│  index.html + script.js + style.css (الواجهة)          │
└──────────────────┬──────────────────────────────────────┘
                   │
┌──────────────────▼──────────────────────────────────────┐
│      طبقة البيانات (Data Layer)                        │
│  manifest.json + Front Matter في ملفات Markdown       │
└──────────────────┬──────────────────────────────────────┘
                   │
┌──────────────────▼──────────────────────────────────────┐
│     طبقة المحتوى (Content Layer)                       │
│  AI-Augmented-Vault/ مع 4 فئات رئيسية                 │
└─────────────────────────────────────────────────────────┘
```

## 2️⃣ المكونات الرئيسية

### 2.1 مكون الواجهة (Frontend)

**الملفات:**
- `index.html` - الهيكل الأساسي
- `script.js` - المنطق والتفاعل
- `style.css` - التنسيق والمظهر

**الوظائف:**
- عرض المحتوى بطريقة منظمة
- البحث والتصفية
- التنقل بين الفئات
- دعم الجوال (Responsive)

### 2.2 مكون البيانات الوصفية (Metadata)

**الملفات:**
- `manifest.json` - قاموس شامل لكل المحتوى
- `generate-manifest.js` - سكريبت لإنشاء البيانات الوصفية

**الوظائف:**
- فهرسة كاملة للمحتوى
- استخراج معلومات من Front Matter
- تحديث تلقائي للتغييرات

### 2.3 مكون المحتوى (Content)

**المجلد الرئيسي:** `AI-Augmented-Vault/`

**الفئات الأربع:**
1. `1-creative-production/` - الإنتاج الإبداعي
2. `2-radical-discourse/` - الخطاب الجذري
3. `3-practical-tech/` - الحلول التقنية
4. `4-knowledge-management/` - إدارة المعرفة

### 2.4 مكون البناء والصيانة (Build & Maintenance)

**الملفات:**
- `build_vault.py` - أداة بناء وصيانة القبو
- `categories.md` - قائمة الفئات

## 3️⃣ هيكل البيانات (Data Structure)

### 3.1 بنية ملف Manifest

```json
{
  "content": [
    {
      "id": "unique-id",
      "title": "عنوان الملف",
      "description": "وصف قصير",
      "category": "1-creative-production",
      "tags": ["tag1", "tag2"],
      "public": true,
      "author": "اسم المؤلف",
      "date": "2026-04-30",
      "filePath": "path/to/file.md",
      "wordCount": 1500,
      "readingTime": 5
    }
  ]
}
```

### 3.2 بنية Front Matter

```yaml
---
title: عنوان الملف
description: وصف قصير يشرح محتوى الملف
category: اسم الفئة
tags: [tag1, tag2, tag3]
public: true
author: اسم المؤلف
date: 2026-04-30
updated: 2026-04-30
difficulty: intermediate
duration: 5
---
```

### 3.3 ترتيب الفئات

```
مستوى 1: الفئات الرئيسية (1, 2, 3, 4)
├── مستوى 2: الأقسام الفرعية
│   └── مستوى 3: المواضيع
│       └── مستوى 4: الملفات (أقصى عمق)
```

## 4️⃣ تدفق البيانات (Data Flow)

### 4.1 دورة القراءة (Read Cycle)

```
المستخدم يفتح الموقع
    ↓
index.html يحمّل script.js + style.css
    ↓
script.js يحمّل manifest.json
    ↓
البيانات تُعرض في الواجهة
    ↓
المستخدم يختار محتوى
    ↓
ملف Markdown يُحمّل
    ↓
المحتوى يُعرض للمستخدم
```

### 4.2 دورة الكتابة (Write Cycle)

```
إضافة ملف Markdown جديد
    ↓
إضافة Front Matter
    ↓
تشغيل: node generate-manifest.js
    ↓
manifest.json يُحدّث
    ↓
الواجهة تُحدّث تلقائياً
    ↓
المحتوى الجديد يظهر
```

## 6️⃣ العمليات الرئيسية (Core Processes)

### 6.1 عملية إنشاء البيانات الوصفية

```javascript
// generate-manifest.js
1. البحث عن جميع ملفات .md في AI-Augmented-Vault
2. قراءة Front Matter من كل ملف
3. التحقق من الحقول المطلوبة (public: true)
4. فرز البيانات حسب الفئة والترتيب
5. إنشاء manifest.json
6. إغلاق البرنامج بنجاح
```

### 6.2 عملية العرض في الواجهة

```javascript
// script.js
1. حمّل manifest.json عند فتح الصفحة
2. نظّم البيانات حسب الفئات
3. أنشئ عناصر HTML ديناميكياً
4. طبّق CSS التنسيق
5. أضف مستمعي الأحداث (Event Listeners)
6. فعّل البحث والتصفية
7. اعرض النتائج
```

## 7️⃣ مستويات الوصول (Access Layers)

### 7.1 طبقة المحتوى العام

```
المحتوى المنشور (public: true)
    ↓
يظهر في manifest.json
    ↓
يظهر في الواجهة
```

### 7.2 طبقة المحتوى الخاص

```
المحتوى الخاص (public: false)
    ↓
لا يظهر في manifest.json
    ↓
متوفر محلياً فقط
```

## 8️⃣ الأمان والخصوصية

### 8.1 نموذج الأمان

```
Frontend:
├── لا توجد بيانات حساسة مكشوفة
├── كل البيانات من الملفات الثابتة
└── لا توجد اتصالات خادم مباشرة

File System:
├── .gitignore لحماية الملفات الحساسة
├── حقوق قراءة/كتابة محدودة
└── نسخ احتياطية منتظمة
```

### 8.2 معايير الجودة

```
✓ لا تخزين كلمات مرور
✓ لا تتبع المستخدمين
✓ لا إعلانات
✓ محتوى موثوق
```

## 9️⃣ التوسع المستقبلي (Scalability)

### 9.1 آلية الإضافة

إضافة محتوى جديد بسيطة جداً:

```
1. أنشئ ملف .md جديد في المجلد الصحيح
2. أضف Front Matter الصحيح
3. اكتب المحتوى
4. شغّل: node generate-manifest.js
5. النتيجة: تحديث تلقائي
```

### 9.2 آلية التطوير

```
يمكن توسيع المشروع بـ:
├── مكونات واجهة جديدة
├── خوارزميات بحث متقدمة
├── دعم لغات جديدة
├── نظام تصنيف متقدم
└── ميزات تعاون
```
