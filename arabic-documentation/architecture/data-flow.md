# تدفق البيانات

## شرح مفصل لكيفية تدفق البيانات عبر النظام

يوضح هذا الملف كيف تتحرك البيانات من الإدخال إلى الإخراج.

---

## 1️⃣ تدفق البيانات الأساسي

### 1.1 مسار القراءة (Read Path)

```
مستخدم يزور الموقع
         ↓
متصفح يطلب index.html
         ↓
خادم يرسل index.html
         ↓
المتصفح يحمّل CSS و JavaScript
         ↓
script.js يحمّل manifest.json
         ↓
manifest.json يُعرض في الذاكرة
         ↓
المستخدم يبحث/يتصفح
         ↓
script.js يُرسل نتائج البحث
         ↓
الواجهة تعرض المحتوى
```

### 1.2 مسار الكتابة (Write Path)

```
مالك المشروع ينشئ ملف .md جديد
         ↓
يضيف Front Matter بالبيانات الوصفية
         ↓
ينقل الملف لمجلد صحيح
         ↓
يشغّل: node generate-manifest.js
         ↓
السكريبت يبحث عن ملفات .md
         ↓
يستخرج Front Matter من كل ملف
         ↓
يتحقق من الحقول المطلوبة
         ↓
ينشئ manifest.json جديد
         ↓
يحفظ الملف
         ↓
المستخدم يعيد تحميل الصفحة
         ↓
المحتوى الجديد يظهر
```

## 2️⃣ مراحل معالجة البيانات

### 2.1 مرحلة الإدخال (Input Stage)

```
┌─────────────────────────────────────┐
│    مستخدم / محرر / آلية             │
└────────────────┬────────────────────┘
                 │
                 ↓
        ┌────────────────┐
        │ ملفات Markdown │
        │ +Front Matter  │
        └────────────────┘
```

**البيانات الأولية:**
- ملفات `.md` نصية عادية
- بيانات وصفية في YAML
- محتوى بتنسيق Markdown

### 2.2 مرحلة المعالجة (Processing Stage)

```
┌────────────────────────────────────────┐
│  generate-manifest.js (المعالج)       │
├────────────────────────────────────────┤
│ 1. البحث عن ملفات                     │
│ 2. قراءة Front Matter                 │
│ 3. التحقق من البيانات                │
│ 4. تنظيم وفرز                         │
│ 5. إنشاء JSON                         │
└────────────────────────────────────────┘
```

**العمليات:**
```javascript
// البحث عن ملفات
readDir('AI-Augmented-Vault')

// استخراج البيانات
parseFrontMatter(file)

// التحقق
validate(data)

// التنظيم
sort(data)
```

### 2.3 مرحلة الإخراج (Output Stage)

```
┌────────────────────────┐
│    manifest.json       │
│    (بيانات منظمة)     │
└────────────┬───────────┘
             │
             ↓
      ┌──────────────┐
      │  الواجهة     │
      │(index.html)  │
      └──────────────┘
```

**البيانات النهائية:**
- ملف JSON منظم
- معلومات كاملة مجهزة للعرض

## 3️⃣ تدفق البيانات التفصيلي

### 3.1 من الملف إلى manifest.json

```
📄 ai-music.md
│
├─ ---
├─ title: الموسيقى المولدة بالذكاء الاصطناعي
├─ description: دليل شامل لإنشاء موسيقى...
├─ category: 1-creative-production
├─ tags: [AI, music, generation]
├─ public: true
├─ author: محمد أحمد
├─ date: 2026-04-30
├─ ---
│
└─ محتوى الملف يبدأ هنا...

          ↓ (معالجة)

📋 manifest.json entry:
{
  "id": "ai-music-1",
  "title": "الموسيقى المولدة...",
  "description": "دليل شامل...",
  "category": "1-creative-production",
  "tags": ["AI", "music"],
  "public": true,
  "author": "محمد أحمد",
  "date": "2026-04-30",
  "filePath": "1-creative-production/audio-generation/ai-music/ai-music.md",
  "wordCount": 3200,
  "readingTime": 12
}
```

### 3.2 من manifest.json إلى الواجهة

```
📋 manifest.json
│
├─ content[0] = {title: "الموسيقى...", ...}
├─ content[1] = {title: "الكتابة...", ...}
├─ content[2] = {title: "الفلسفة...", ...}
└─ ...

          ↓ (عرض في الواجهة)

🌐 index.html + script.js
│
├─ <div class="article">
│   ├─ <h3>الموسيقى...</h3>
│   ├─ <p>وصف...</p>
│   ├─ <span class="tags">AI, music</span>
│   └─ <a href="...">اقرأ</a>
├─ <div class="article">
│   └─ ...
└─ ...
```

## 4️⃣ دورة تحديث البيانات

### 4.1 الدورة الكاملة

```
الساعة 00:00
    ↓
المستخدم ينشئ ملف جديد
    ↓
يضيف Front Matter
    ↓
الساعة 06:00
    ↓
يشغّل: npm start (أو node generate-manifest.js)
    ↓
النظام يبحث عن التغييرات
    ↓
يستخرج البيانات
    ↓
ينشئ manifest.json جديد
    ↓
ينشر التغييرات
    ↓
الساعة 06:05
    ↓
المستخدمون يشاهدون المحتوى الجديد
```

### 4.2 تحديثات الملفات الموجودة

```
ملف قديم (ai-music.md)
    ↓
المالك يعدّل المحتوى
    ↓
يشغّل generate-manifest.js
    ↓
السكريبت يكتشف التعديلات
    ↓
يحدّث manifest.json
    ↓
الواجهة تحدّث البيانات
    ↓
المستخدمون يرون الإصدار الجديد
```

## 5️⃣ تدفق البيانات الوظيفية

### 5.1 البحث والتصفية

```
مستخدم يكتب: "موسيقى"
         ↓
script.js يبحث في manifest.json
         ↓
يجد جميع الإدخالات التي تحتوي على "موسيقى"
         ↓
يفلترها حسب الفئة/الوسم
         ↓
يرتبها حسب الصلة
         ↓
يعرض النتائج للمستخدم
```

### 5.2 التنقل والوصول

```
المستخدم ينقر على فئة
         ↓
script.js يحمّل جميع المقالات في الفئة
         ↓
يرتبها حسب التاريخ/الاسم
         ↓
يعرضها في قائمة
         ↓
المستخدم ينقر على مقال
         ↓
الملف .md يحمّل
         ↓
المحتوى يُعرض
```

## 7️⃣ كفاءة تدفق البيانات

### 7.1 استراتيجيات التحسين

**التخزين المؤقت (Caching):**
```javascript
// حفظ manifest.json محلياً
localStorage.setItem('manifest', JSON.stringify(data))

// استخدام البيانات المحفوظة
const cached = localStorage.getItem('manifest')
```

**التحميل الكسول (Lazy Loading):**
```javascript
// تحميل المحتوى عند الطلب فقط
if (userScrolledToBottom) {
  loadMoreContent()
}
```

**الضغط (Compression):**
```javascript
// ضغط البيانات
const compressed = gzip(manifest)

// فك الضغط عند الحاجة
const decompressed = gunzip(compressed)
```

### 7.2 مراقبة الأداء

```
قياس الأداء:
├─ وقت تحميل manifest.json
├─ وقت معالجة البحث
├─ وقت عرض النتائج
└─ حجم البيانات المنقولة
```

## 9️⃣ متطلبات جودة البيانات

### 9.1 التحقق من الصحة

```
كل ملف يجب أن يحتوي على:
✓ title (عنوان)
✓ description (وصف)
✓ category (فئة صحيحة)
✓ tags (وسوم)
✓ public (true/false)
✓ author (اسم المؤلف)
✓ date (تاريخ صحيح)
```

### 9.2 معايير الصيغة

```
Front Matter:
├─ YAML صحيح
├─ اقتباسات صحيحة
├─ تنسيق صحيح
└─ بدون أخطاء إملائية

Markdown:
├─ روابط صحيحة
├─ تنسيق صحيح
├─ بدون أخطاء إملائية
└─ محتوى واضح
```

## 🔟 مثال عملي شامل

### 10.1 من البداية إلى النهاية

```
مرحلة 1: الإنشاء
├─ نشاء ملف: songwriting.md
├─ إضافة Front Matter:
│  - title: فن الكتابة الغنائية
│  - description: دليل شامل...
│  - category: 1-creative-production
│  - public: true
│  └─ tags: [songwriting, music, creative]
└─ نسخه إلى 1-creative-production/audio-generation/songwriting/

مرحلة 2: المعالجة
├─ شغّل: node generate-manifest.js
├─ السكريبت يقرأ الملف
├─ يستخرج البيانات الوصفية
├─ يتحقق من الصحة
└─ ينشئ manifest.json جديد

مرحلة 3: العرض
├─ المستخدم يفتح الموقع
├─ script.js يحمّل manifest.json
├─ يعرض قائمة بالمحتوى الجديد
├─ تظهر "فن الكتابة الغنائية" في القائمة
└─ المستخدم ينقر وقراءة المقال

مرحلة 4: التحديث
├─ المالك يعدّل الملف
├─ يشغّل generate-manifest.js مجدداً
├─ manifest.json يحدّث
└─ نسخة جديدة تظهر
```

## ملخص تدفق البيانات

| المرحلة | المدخل | المعالج | المخرج |
|--------|--------|---------|--------|
| الإنشاء | نص عادي | محرر نصوص | ملف .md |
| الاستخراج | .md files | gray-matter | بيانات YAML |
| التجميع | بيانات متعددة | generate-manifest.js | manifest.json |
| العرض | JSON | script.js | واجهة ويب |
